# ValueCommit 웹 데모 설명: 무엇을 / 어떻게 만들었나 / 근거

2026.06.11 · 지도 미팅용

---

## 0. 한 줄 요약

사용자가 직접 말하지 않은 hidden intention(숨은 쇼핑 기준)을 대화와 행동(고른 것 / 버린 것)에서 읽어내고,
그 해석을 사용자가 보고 고칠 수 있게 돌려주는 쇼핑 에이전트.
여기서 ontology는 대화에서 찾은 의도, 그 의도가 가리키는 가치, 판단의 근거가 된 발화를 서로 연결해 쌓아가는 지식 그래프를 말함.
이론에서 출발한 가치 구조와 데이터에서 새로 찾은 기준을 이 그래프 하나로 합쳤고,
이 구조가 실제로 작동하는지 합성 대화 10건으로 처음 확인했다.

---

## 1. 설계 골격

- Top-down 기반 초기 설계
  - 소비자 가치 이론, hedonic/utilitarian value, PSCon 시나리오를 기반으로 초기 ontology와 user agent를 설계
  - 가치 축을 데이터에서 임의로 만들지 않고 검증된 소비자 가치 이론에서 가져옴, 해석이 이론적으로 정당하고, 사용자·시나리오가 달라도 같은 축으로 비교 가능
- Bottom-up 기반 확장
  - 사용자가 고른 상품과 버린 상품의 차이를 비교해, 사용자가 말하지 않은 기준을 찾아냄
  - 사전에 정의하지 못한 hidden intention feature로 활용할 수 있음
- Human-in-the-loop
  - 새로 발견된 hidden intention이 기존 ontology와 충돌하거나 모호할 때, 사용자에게 보여주고 수정하게 함
- Hybrid ontology
  - 이론에서 온 가치 축과, 데이터에서 찾은 feature를 같은 그래프 안에 둠
  - top-down만 쓰면 놓치는 패턴(새 기준)과, bottom-up만 쓰면 부족한 일반화(공통 구조)를 같이 포착하고자 함

---

## 2. 가치를 두 층으로 본다 (global / local)

같은 "신중하게 고른다"는 행동도 출처가 다를 수 있다. 원래 신중한 사람이라 그런 경우(그 사람의 가치)와,
이번이 선물이라 그런 경우(그 상황의 동기)를 구분해야 해석이 정확해진다. 그래서 두 층으로 나눈다.

- global = 그 사람의 가치 (TCV, Sheth·Newman·Gross 1991)
  - Functional: 성능·내구성·가격 대비 효용 ("실용적인 이점을 주는가")
  - Social: 사회적 이미지·체면·관계 ("이걸 쓰는 게 내 이미지를 좋게 만드는가")
  - Emotional: 감정·안심 ("긍정적인 감정을 주는가, 불안을 줄여주는가")
  - Epistemic: 호기심·새로움·정보 탐색 ("호기심을 자극하는가")
  - Conditional: 특정 상황에서만 생기는 효용 ("이 상황이라서 특히 유용한가", 계절 상품·비상용품 등)
  - 그 사람의 고유한 특성에 가까워 잘 변하지 않음 → 여러 세션·시나리오에 걸쳐 한 사람에게 쌓아감
- local = 그 상황의 동기 (Hedonic / Utilitarian)
  - Utilitarian = 목적 달성(필요한 걸 효율적으로 사기), Hedonic = 쇼핑 경험 자체의 즐거움(구경, 기분전환, 자기보상)
  - 사는 상황·상품에 따라 바뀜 → 한 시나리오 안에서만 적용
  - 측정할 때는 더 잘게 나눔: Hedonic을 Arnold·Reynolds(2003)의 6가지 즐거움(모험·기분전환·선물·득템·사교·트렌드)으로 나누고, 여기에 Utilitarian을 더해 7가지 동기로 측정 (데모 방사형의 동기 7축이 이것)
  - 에이전트는 이 동기를 설문으로 묻지 않고, 해당 설문 문항의 내용을 반영한 자연스러운 질문으로 대화 중에 끌어냄 ("이번엔 새로운 걸 발견하는 재미로 둘러보고 싶으세요, 정해둔 걸 찾으세요?" 같은 식)

가치를 측정하는 두 가지 원칙

- 점수가 아니라 등급으로 판단함: 발화를 보고 0.7 같은 임의 숫자를 매기게 하면 같은 발화라도 매번 다른 숫자가 나옴. 대신 정해진 등급(직접 말함 / 강하게 추론됨 / 약한 힌트)으로만 판단하게 하고, 근거가 된 발화를 반드시 함께 제시하게 함. 근거 인용이 없으면 그 판단은 버림. 이렇게 하면 다른 사람(또는 다른 모델)이 따로 판정해도 결과를 비교할 수 있어, 측정의 신뢰도를 확인할 수 있음(평가자 간 일치율 κ)
- '숨었다'의 기준은 출처: 어떤 기준이 숨은 의도인지 아닌지는 그 기준이 어디서 나왔는지로 정함. 사용자가 직접 말했으면(explicit) 드러난 것, 직접 말한 적은 없는데 행동·반응에서만 나타났으면(implicit/latent) 숨은 것. 추론 결과에 사후로 딱지를 붙이는 게 아니라, 근거의 출처에서 자동으로 정해지는 구조

## 3. 합성이 어떻게 돌아가나 (페르소나 → GT, 에이전트 3종)

한 번의 합성은 이렇게 돈다: 페르소나에서 정답(GT)을 만들고 → user agent가 그 사람이 되어 service agent와 대화하며 →
service agent가 말과 행동만으로 그 사람의 가치를 복원한다. 핵심은 GT를 user agent만 알고 service agent에는 숨기는 것이다.
정답을 알고 시작하기 때문에, 시스템이 숨은 의도를 얼마나 복원했는지 채점할 수 있다.

페르소나에서 GT(정답) 만들기

- 재료는 Nemotron-Personas-Korea: 한국인 가상 인물 데이터셋으로, 인구통계와 직업·취미·가족·가치관 같은 삶의 서사가 적혀 있음. 쇼핑 취향은 직접 안 적혀 있음
- LLM이 이 서사에서 쇼핑 가치를 미루어 짐작함 (기재된 내용을 그대로 옮기는 게 아니라 추론하는 것)
- 정해진 항목을 채우도록 함: TCV 5가치 각각의 강도(dominant 지배적 / present 있음 / trace 흔적만), 두드러진 동기, 숨은 의도 2~3개, 말투, 어울리는 쇼핑 시나리오. 각 판단에 서사 근거를 함께 달도록 함
- 실제 예 (가상 인물 조광호): "포항에서 철강 설비를 다뤄온 꼼꼼한 기술자, 지금은 당뇨를 관리하며 지냄"이라는 서사 → 도출된 GT: Functional 지배적, 숨은 의도 "고장 없이 오래 쓸 수 있는지 꼼꼼히 따짐", "건강에 무리가 없는지 확인함", "복잡한 기능보다 조작이 쉬운 것을 선호함"
- 한쪽으로 치우치지 않도록 프롬프트로 제약함: dominant를 남발하지 말 것, Functional을 습관적으로 우선하지 말 것(누구나 실용은 어느 정도 따지므로), 서사에 체면(Social)·새로움(Epistemic)·상황 기준(Conditional)의 단서가 있으면 그 축을 먼저 검토할 것
- 페르소나 10명은 성별·연령대가 겹치지 않게 골라 스펙트럼을 넓힘
- 형식이 안 맞으면(5축 등급 누락·시나리오 불일치·숨은 의도 없음) 버리고 다시 생성함. 통과한 프로필은 파일로 저장해 사람이 직접 검토·수정 가능
- 이 프로필이 GT(정답). 대화 시작 전 고정되고, service agent에는 노출하지 않음

대화에 쓰는 상품

- 대화에 등장하는 상품은 아직 자체 제작 카탈로그 (네이버 쇼핑 데이터 연동은 예정)
- 상품마다 단서(cue)를 달아둠. 예: "39,900원 초경량 모델 = 가격이 매우 낮음 + 인기 많음 + 장기 사용 리뷰 적음 + 판매자 등급 낮음"
- user agent는 이 단서에 자기 GT대로 반응함 (체면을 중시하는 사람은 너무 싼 상품을 거절, 실패가 불안한 사람은 장기 리뷰가 적은 상품을 회피)
- service agent는 거꾸로, 어떤 단서가 선택과 거절을 갈랐는지를 보고 숨은 기준을 역추적함

user agent: 그 사람으로 연기

- GT를 받아 페르소나가 되어 한국어로 대화. 예를 들어 조광호는 "3년 쓴 스마트워치가 고장 났는데, 좋은 걸로 바꾸려고 합니다"로 시작함
- 실제 쇼핑 채팅처럼 짧게 말하고, 한 턴에 새 조건은 하나씩만. 요구사항을 처음부터 전부 말하지 않고, 에이전트가 물어봐야 조금씩 답함
- 숨은 의도는 직접 말하지 않음. 에이전트가 관련 질문을 하거나 상품에 반응할 때만 간접적으로 드러남 ("좀 싸 보여서요"처럼 체면을 대놓고 말하지 않는 식)
- 상품마다 좋아요/싫어요/자세히보기로 반응하고, 마음에 들면 구매 의사를 밝히고, 흥미가 떨어지면 탐색을 멈춤

service agent: 말과 행동만으로 추론

- GT를 모름. 대화와 행동(좋아요/싫어요·선택/거절)만 보고 추론
- 매 턴 다음을 돌림: 발화에서 의도 후보 찾기 → 같은 의도끼리 병합 → 각 의도를 TCV 가치(global)에 연결하고 상황 동기(local)도 함께 감지 → 기존 해석과 충돌하는지 점검 → 가치 상태 갱신
- 가치 점수의 계산: 의도 하나가 어떤 축을 뒷받침하면 그 축이 조금 오르고, 같은 축을 가리키는 의도가 여러 개 쌓일수록 점수가 1에 가까워지는 방식(noisy-OR). 근거가 많을수록 확신이 커지는 직관을 그대로 수식화한 것
- 충돌 감지: 새 신호가 앞선 가정과 어긋날 때 잡아냄. 예: 처음엔 "저렴한 게 좋아요" → 나중에 너무 싼 상품을 거절 → "가격이 싸야 한다"와 "너무 싸면 불안하다"가 충돌 → 확인 카드로 사용자에게 어느 쪽이 맞는지 물어봄
- 추론한 내용은 사용자가 확인·수정할 수 있게 돌려줌. 일방적 추론이 아니라 함께 맞춰가는 과정

judge agent: 추론을 따로 검증

- service agent가 "A라서 B" 같은 인과 추론을 만들면(예: "무릎이 안 좋아서 전문 러닝화를 피한다"), 별개의 LLM이 그 주장이 실제 발화로 뒷받침되는지 대조해 판정
- 만든 쪽이 스스로 채점하면 관대해지기 쉬우므로, 만드는 역할과 검증하는 역할을 분리한 것
- 내용은 고치지 않고 검증 결과만 표시(지지 / 강등). 권위 순서는 사용자 > judge > service agent (사용자가 직접 확인해준 것은 judge도 못 뒤집음)
- 대화를 느리게 만들지 않도록 응답을 보낸 뒤 별도로 실행함

## 4. 데모에서 보는 것

시뮬레이션: 합성 대화 검수

- 페르소나 카드 목록이 먼저 보임. 카드마다 넣은 가치와 복원된 가치, 일치 여부(✅/❌)가 표시되고, 불일치가 위로 오게 정렬해 검토가 필요한 사례부터 눈에 띄게 함
- 카드를 누르면: 왼쪽에 합성 대화 전체(발화 + 좋아요/싫어요/구매 행동 기록), 오른쪽에 그 사람에게 넣은 숨은 의도(GT)와 시스템이 복원한 결과가 나란히 표시됨
- ✅일치 / ❌불일치의 기준: 넣은 dominant 가치가 복원 상위 2축에 들어오면 일치 (지금은 가치 이름만 비교함. 이름은 같아도 근거가 다른 경우까지 가려내는 것은 다음 단계 과제)
- 방사형 그래프: 축마다 그 가치가 얼마나 강하게 복원됐는지를 채움 정도로 보여줌. 5축 = TCV 가치, 7축 = 쇼핑 동기. 축을 누르면 어떤 의도·발화가 그 축을 뒷받침했는지 분해해서 보여줌

PSCon: 같은 프레임워크로 실데이터까지

- PSCon은 실제 쇼핑 대화 648건이 담긴 공개 데이터셋. 시나리오를 잡을 때 참고했고, 우리 프레임워크의 비교 기준으로도 씀
- 위에서 만든 분석 프레임워크(대화 → 의도 → 가치)를 그대로 이 648건에 적용
- 한 대화씩 재생하면서, 턴이 진행될수록 가치·동기 축이 어떻게 채워지는지 추적할 수 있음
- 648건 전부 분석(실패 0) → 거래형 대화라 Utilitarian(목적 달성) 동기가 압도적이고 숨은 의도는 거의 나타나지 않음
- 의미: 기존 실데이터엔 우리가 보려는 hidden intention이 드물다 → 그래서 합성이 필요함. 동시에 합성 데이터와 실데이터를 같은 척도로 측정할 수 있다는 점도 보여줌

## 5. 첫 합성 결과 (페르소나 10명)

- 10건 모두 중간에 끊기지 않고 끝까지 대화 · 평균 4.8턴 · 6건은 구매 의사를 밝히고 종료, 4건은 구매 없이 둘러보다 종료
- 고른/버린 상품 쌍 27개에서 → 미리 정의하지 못한 숨은 기준 후보 8개를 찾음
  - 찾는 방식: 여러 대화에서 비슷한 선택/거절 패턴이 반복되면 하나의 기준 후보로 묶음. 예를 들어 "너무 싼 상품을 거절한 이유"가 여러 사람에게서 반복되자 "선물 가격 하한(체면 가격대)"이라는 기준 후보가 됨
  - 그 밖의 예: "신뢰가 안 가는 셀러는 회피", "흔하지 않은 선물의 특별함" (모두 처음 이론 축에는 없던 것)
- 넣은 dominant 가치 ↔ 복원: 9/10 일치 / 안 맞은 1건(Epistemic을 넣었는데 못 복원)은 검수 화면 상단에 자동으로 표시됨
- 숨은 의도가 말이 아니라 행동으로 드러난 예
  - "선물이지만 20만원 넘으면 부담돼요" → 가격 부담을 직접 드러내지 않으면서 거절 이유로만 표현
  - "판매자가 생소해서 좀 더 자세히 보고 싶어요" → 구매 실패에 대한 불안이 자세히보기 행동으로 나타남
  - 처음엔 "예쁜 디자인이면 더 좋죠"라고 말하던 사람이, 무릎이 안 좋다는 얘기를 반복하다 결국 디자인 대신 편한 신발을 구매 → 시스템이 기준이 바뀐 것을 감지하고 확인을 요청함
