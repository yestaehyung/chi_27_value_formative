# 상품 라벨링 → 태그 기반 단계형 추천 (Option 1) 구현 계획

**작성일:** 2026-06-17
**배경:** §14.2 가중합이 근거 없음 + 임베딩은 제목-only 노이즈로 실패 + 카테고리 내 수식어("운동용/브이넥/울")가 무시됨.
관련: `2026-06-16-formative-study-deploy-design.md` §5, 코딩스펙 §14.2/§36

## 0. 목표 (왜)

원시 제목 대신 **LLM으로 상품을 구조화 태그/속성으로 라벨링**하여:
1. 카테고리 내 수식어가 실제로 작동 (운동용→운동/오픈형 태그 필터)
2. 추천 **근거를 보여줌** (§36 — 매칭 태그가 곧 evidence; 임베딩 코사인은 불투명해서 부적합)
3. 비어 있던 `hidden_intention_fit` 속성을 채워 **근거 없는 가중합 의존을 줄임**(단계형 필터로 대체)

## 1. 데이터 스키마 (상품당 채움 — 기존 필드 재사용)

```jsonc
// 라벨러(sub-agent) 출력 — 상품당
{
  "id": "nv_...",
  "tags": ["오픈형","운동","노이즈캔슬링"],   // 허용 vocab(config/tag_taxonomy.json) 부분집합 — 구조화 신호
  "offCategory": false,                       // true = 이 카테고리의 진짜 상품 아님(정화에서 제거; LLM 판단)
  "shortDesc": "달릴 때 빠지지 않는 오픈형 노이즈캔슬링 무선 이어폰"  // 제목 근거 1문장(향후 임베딩용)
}
// 병합(merge_labels.py) 시 attributes는 메타(categoryId/domain)만 유지 — 라벨러 자유형 attribute 키는 폐기.
// 정화(clean_pool.py)는 offCategory 플래그로 거름(없으면 regex 폴백).
```

## 2. 태그 택소노미 (카테고리별 **허용 vocab** — 일관성의 핵심)

LLM이 free-form이 아니라 **이 목록에서만** 고른다(매칭 일관성). 카테고리별 8~12개. 초안:
- 무선이어폰: `오픈형·커널형·넥밴드 / 운동 / 게이밍 / 노이즈캔슬링 / 방수 / 통화특화 / 저지연 / 프리미엄`
- 노트북: `게이밍 / 사무용 / 영상편집 / 경량휴대 / 고성능 / 대화면 / 2in1`
- 원피스: `캐주얼 / 오피스 / 하객·포멀 / 미니·미디·롱 / 패턴 / 무지`
- 니트·가디건: `브이넥·라운드·터틀 / 울·캐시미어·면·아크릴 / 얇음·두꺼움 / 오버핏·슬림 / 베이직·디자인`
- 코트·패딩·자켓: `패딩·코트·자켓·점퍼 / 숏·롱 / 캐주얼·포멀 / 방한`
- (나머지 카테고리도 동일 형식)

## 3. 라벨링 프롬프트 (deepseek, 오프라인)

입력: `{title, category, 허용태그목록}` → 출력 JSON `{tags, attributes, description, unknown}`.
**규율:** 제목에 명시·강하게 시사된 것만. **불확실은 넣지 말 것(환각 금지)** → `unknown`에 표기. 한국어.

## 4. 단계(Phases)

| # | 단계 | 산출물 | 게이트 |
|---|---|---|---|
| 0 | 택소노미 확정 | `config/tag_taxonomy.json` (카테고리별 태그 vocab) | — |
| 1 | 라벨러 스크립트 | `scripts/label_products.py` (asyncio 동시성·재개가능) | — |
| 2 | **프로토타입 20개**(이어폰+니트) | 태그품질·환각률 리포트 | ✅ 품질 OK여야 진행 |
| 3 | 전체 650 라벨링 + QA | `products_stratified.labeled.json` | 스팟체크 |
| 4 | **검색 단계형 재작성** | `search.py` (가중합 제거) | "운동용/10만원" 바인딩 확인 |
| 5 | 가치모델·evidence 배선 | `hidden_intention_fit`이 태그 읽음 / 카드에 태그 노출 | — |
| 6 | 평가 | 수식어 질의 3종 재현 | — |

**4단계 단계형 검색(가중합→필터):**
```
1) 관련성 게이트  : 카테고리 + 질의 토큰 (후보 자격)
2) 수식어→태그 필터: 질의 수식어 추출(기존 topic 추출 + 동의어맵 운동→운동·러닝·오픈형) → 태그/속성 매칭
3) trade-off 다양화: 저가/고신뢰/독특 버킷 1개씩
4) tie-break       : 리뷰수·구매수
→ 튜닝 가중치 0. 각 단계 개별 정당화.
```

## 5. 위험·주의

- **환각**: 제목에 없는 속성 추론 금지 → `unknown`. 프로토타입(2단계)에서 환각률 측정 후 진행.
- **비용**: 650 deepseek콜(오프라인 1회, 저렴). 동시성으로 수분 내.
- **다른 세션 충돌 회피**: 입력=`products_stratified.json`, 출력=`.labeled.json`(별도 파일). 이미지/NAVER category는 다른 필드라 나중에 머지(무충돌).
- **태그 vocab 정합**: 카테고리별 분리로 "이어폰에 브이넥" 같은 오태깅 방지.

## 6. subagent / 병렬화 (정직하게)

- **라벨링 본체는 스크립트(deepseek 루프)** — Claude 서브에이전트 650개로 돌리는 건 비용·부적합. 스크립트가 정답.
- 서브에이전트가 **유효한 곳**: (a) 카테고리별 태그 택소노미 초안 병렬 작성, (b) 라벨러 스크립트 ↔ 검색 재작성 동시 진행, (c) 라벨 품질 검수 에이전트.
- 본 작업은 *반복 검증이 중요*(환각·태그 정합)해서, 핵심은 메인 스레드 + 체크포인트로 가고 독립 조각만 병렬화 권장.

## 7. 시작점

Phase 0(택소노미) + Phase 1(스크립트) → **Phase 2 프로토타입 20개로 게이트**. 품질 확인 후 전체.
